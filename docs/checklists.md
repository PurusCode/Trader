# Чек-лист тестирования: Система алгоритмического трейдинга

## 1. Позитивные сценарии (Happy Path)
- [ ] **Авторизация:** Успешное подключение к Tinkoff Invest API с валидным токеном.
- [ ] **Получение данных:** Корректное получение котировок и свечей для списка FIGI.
- [ ] **Расчет индикаторов:** Верный расчет RSI и ATR (сверка с эталоном TA-Lib).
- [ ] **Сигнал на покупку:** Срабатывание сигнала при `RSI < 32` и росте значения.
- [ ] **Исполнение ордера (Buy):** Успешная отправка заявки, статус `EXECUTED`.
- [ ] **Фиксация сделки:** Запись данных в файл `akcii_kupil.txt` и логирование.
- [ ] **Уведомление:** Отправка сообщения в Telegram о покупке.
- [ ] **Сигнал на продажу:** Срабатывание при `RSI > 60` или достижении Take-Profit.
- [ ] **Исполнение ордера (Sell):** Успешная продажа, зачисление средств.
- [ ] **Фиксация прибыли:** Запись результата в `trade_results.xlsx`, отправка отчета в Telegram.

## 2. Негативные сценарии и валидация
- [ ] **Недостаточно средств:** Попытка покупки при балансе < стоимости лота → Ошибка, ордер не создан.
- [ ] **Рынок закрыт:** Торговая сессия завершена → Корректная обработка ошибки API.
- [ ] **Невалидный FIGI:** Запрос по несуществующему инструменту → Обработка исключения, отсутствие падения.
- [ ] **Ошибка сети (Timeout):** Потеря соединения → Автоматический retry (3 попытки), логирование.
- [ ] **Частичное исполнение:** Куплено меньше лотов, чем запрошено → Пересчет средней цены.
- [ ] **Дублирование сигнала:** Повторный сигнал на покупку той же акции без продажи → Блокировка второй покупки.

## 3. Бизнес-логика и финансы
- [ ] **Расчет лотности:** Формула `Кол-во лотов = Бюджет / (Цена * Лотность)` работает верно.
- [ ] **Stop-Loss / Take-Profit:** Динамический расчет на основе ATR корректен.
- [ ] **Округление:** Итоговые суммы округляются до копеек согласно правилам биржи.
- [ ] **Временные зоны:** Время сделок в логах соответствует MSK/UTC.

## 4. Нефункциональные требования
- [ ] **Производительность:** Цикл проверки всех тикеров укладывается в интервал свечи (5 мин).
- [ ] **Надежность:** Работа 24/7 без утечек памяти (проверка после 24 часов).
- [ ] **Восстановление (Recovery):** Перезапуск бота восстанавливает список открытых позиций из файла.
- [ ] **Безопасность:** Токены не захардкожены в коде (используются env-переменные).
