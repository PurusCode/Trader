# Тест-кейсы: Система алгоритмического трейдинга

### TC-001: Успешная покупка акций при срабатывании сигнала RSI
| Поле | Описание |
| :--- | :--- |
| **ID** | TC-TRADING-001 |
| **Название** | Покупка акций при падении RSI ниже 32 |
| **Предусловия** | Баланс достаточен, акция не куплена ранее, бот запущен. |
| **Шаги** | 1. Сымитировать данные: RSI упал до 31.5 и начал расти.<br>2. Дождаться цикла проверки.<br>3. Проверить логи и файл `akcii_kupil.txt`.<br>4. Проверить портфель в API брокера.<br>5. Проверить Telegram-канал. |
| **ОР** | 1. Ордер отправлен и исполнен.<br>2. Акция записана в файл.<br>3. В портфеле появилась позиция.<br>4. В Telegram пришло сообщение «🔵 #Сигнал на покупку». |

### TC-002: Обработка ошибки «Недостаточно средств»
| Поле | Описание |
| :--- | :--- |
| **ID** | TC-TRADING-002 |
| **Название** | Отмена покупки при недостатке денежных средств |
| **Предусловия** | Баланс меньше стоимости 1 лота целевой акции. Сигнал на покупку есть. |
| **Шаги** | 1. Сформировать сигнал на покупку.<br>2. Запустить функцию `buy()`.<br>3. Проверить логи.<br>4. Проверить историю ордеров. |
| **ОР** | 1. В логе запись: «Не хватает денег для покупки».<br>2. Запрос на биржу НЕ отправлен.<br>3. Файл позиций не изменен.<br>4. Уведомлений нет. |

### TC-003: Восстановление состояния после перезапуска
| Поле | Описание |
| :--- | :--- |
| **ID** | TC-TRADING-003 |
| **Название** | Корректное восстановление списка акций после рестарта |
| **Предусловия** | В файле `akcii_kupil.txt` есть запись о купленной акции `LKOH`. |
| **Шаги** | 1. Принудительно остановить бота.<br>2. Запустить бота заново.<br>3. Дождаться следующего цикла проверки цены для `LKOH`. |
| **ОР** | 1. Бот считал файл при старте.<br>2. Внутренний список позиций содержит `LKOH`.<br>3. Бот мониторит цену для продажи, не пытаясь купить снова. |
